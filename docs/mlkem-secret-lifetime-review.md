## Runtime Limits

- Swift `Data` and `[UInt8]` copies cannot guarantee immediate zeroization after
  ARC lifetime ends.
- Kotlin `ByteArray` copies cannot guarantee zeroization across ART, GC, and
  compiler optimizations.
- C# `byte[]` copies cannot guarantee zeroization across GC and array copies
  unless explicit clearing is added and reviewed.
- Existing APIs expose exportable private-key representations. This is useful
  for package parity and migration, but remains a production risk unless the
  caller has key-lifecycle controls.

## Deterministic Seed Boundaries

- Swift primitive `keypairDerand` and `encapsulateDerand` are package-internal,
  but Swift also exposes a public seed representation initializer for migration.
- Android deterministic keypair and encapsulation helpers are internal at the
  package API boundary; tests exercise them from package test sources.
- C# deterministic keypair and encapsulation helpers are internal and exposed to
  tests through the test project relationship.

## Logging And Diagnostics

No primitive API logs private seeds, secret keys, shared secrets,
encapsulation coins, or incremental encapsulation secrets. Benchmark harnesses
emit aggregate JSON metrics and may use a local accumulator to prevent dead-code
elimination, but they do not print secret byte values.

## Findings

### SL-001: Managed Zeroization Is Not Guaranteed

- Severity: medium
- Status: open
- Evidence: secret material is copied through managed arrays/Data and returned
  to callers.
- Required closure: reviewer accepts residual risk for production fallback or
  production fallback remains fail-closed.

### SL-002: Exportable Private Representation Contains Seed Material

- Severity: medium
- Status: open
- Evidence: private-key representation APIs carry enough material to recreate
  the private key.
- Required closure: production callers must have key-lifecycle controls accepted
  by a reviewer, or fallback remains disallowed for production.

### SL-003: Swift Seed Import Is Public

- Severity: medium
- Status: open
- Evidence: Swift `PrivateKey` exposes a public seed representation initializer
  for migration.
- Required closure: reviewer accepts the migration boundary and caller lifecycle
  requirements for production fallback, or fallback remains fail-closed.

### SL-004: Secret Material Is Not Intentionally Logged

- Severity: informational
- Status: source reviewed
- Evidence: source scan finds no primitive logging of secret values.
